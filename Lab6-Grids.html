Introduction

This lab explores the generative logic and visual expression of the Cantor Set through its original construction and two creative variations. First introduced by German mathematician Georg Cantor in 1883 as a counterexample in set theory (Dauben, 1990), the Cantor Set challenged traditional notions of continuity and laid the groundwork for modern fractal geometry. Beginning with the classical middle-third removal, this study then modifies the rule to preserve alternate segments and finally introduces randomness to disrupt symmetryâ€”demonstrating how small changes in recursive logic can yield vastly different structural and visual outcomes.

1. Animating with Wolfram Mathematica

The Python script below visualizes the Cantor Set through recursive geometric construction and plotting. It begins by importing NumPy and Matplotlib to handle numerical operations and graphical rendering. Initial parameters define the horizontal span, vertical positioning, bar height, spacing between levels, and recursion depth. The cantor_set function implements the fractal logic: at each level, it divides a line segment into three equal parts, removes the middle third, and recursively applies the same operation to the remaining left and right thirds, shifting them downward and reducing the recursion depth until reaching the base case. The output, a list of segments, is converted into a NumPy array for efficient plotting. Each segment is drawn as a black horizontal bar using fill_between, and the result is a stacked, self-similar structure representing the Cantor Set. The axes are hidden to emphasize the fractal pattern, and the plot is displayed in a clean format. 


# Import libraries for math and plotting
import numpy as np
import matplotlib.pyplot as plt

# Set initial parameters
x_start = 0       # Starting x-coordinate of the first bar
x_end = 3         # Ending x-coordinate of the first bar
y_position = 0    # Starting y-coordinate (height of the first bar)
bar_height = 5    # Height of each bar
spacing = 10      # Space between bars (distance downwards)
depth = 7         # Number of levels to draw

# Define a function to create the Cantor Set pattern
def cantor_set(x_start, x_end, y_position, depth):
    if depth == 0:
        return []

    # Left and right segments for the current bar
    left_segment = [[x_start, x_start + (x_end - x_start) / 3, y_position]]
    right_segment = [[x_end - (x_end - x_start) / 3, x_end, y_position]]

    # Recursively generate segments for the next levels
    left_recursive = cantor_set(x_start, x_start + (x_end - x_start) / 3, y_position - spacing, depth - 1)
    right_recursive = cantor_set(x_end - (x_end - x_start) / 3, x_end, y_position - spacing, depth - 1)

    # Combine the current segments with the results from recursive calls
    return left_segment + right_segment + left_recursive + right_recursive

# Generate Cantor Set segments data
segments = np.array(cantor_set(x_start, x_end, y_position, depth))  # Convert to an array for easy plotting

# Plot the Cantor Set
plt.figure(figsize=(25, 25))

# Draw each segment as a black bar
for x_start, x_end, y in segments:
    plt.fill_between([x_start, x_end], [y] * 2, [y - bar_height] * 2, color="black")

# Hide the axes for a clean look
plt.axis('off')
plt.show()

The image below depicts the original Cantor Set, constructed by recursively removing the middle third of a horizontal segment at each level, resulting in a self-similar fractal composed of increasingly fragmented line segments. This geometric process visually embodies the foundational ideas of set theory and early 20th-century explorations into the nature of the infinite and discontinuous (Dauben, 1990).

2. Variations & Analysis

These two variations of the Cantor Set explore how altering deterministic rules can produce distinct visual and structural outcomes in fractal generation. By modifying the number of preserved segments or introducing randomness, each version highlights different dimensions of complexity, symmetry, and visual rhythm within recursive systems. Below, there are two variations of the Cantor Set, with their respective code described in some detail.

2.1. Fractal Tridents

In this version of the code, the traditional Cantor Set construction is modified by dividing each segment into five equal parts instead of three and removing the second and fourth parts rather than the middle third. The recursive function now preserves the 1st, 3rd, and 5th segments at each level, resulting in a more complex and denser fractal pattern that retains three sub-bars per iteration rather than two. Additionally, the total segment width (x_end) and vertical spacing between levels have been adjusted to accommodate the new structure. 


import numpy as np
import matplotlib.pyplot as plt

# Parameters
x_start = 0
x_end = 5
y_position = 0
bar_height = 5
spacing = 5
depth = 5

# Modified Cantor Set: split into 5, remove 2nd and 4th
def modified_cantor_set(x_start, x_end, y_position, depth):
    if depth == 0:
        return []

    length = x_end - x_start
    piece = length / 5

    # Keep 1st, 3rd, and 5th parts
    segments = [
        [x_start, x_start + piece, y_position],
        [x_start + 2 * piece, x_start + 3 * piece, y_position],
        [x_start + 4 * piece, x_end, y_position]
    ]

    # Recursively generate for the 3 kept segments
    recursive = (
        modified_cantor_set(x_start, x_start + piece, y_position - spacing, depth - 1) +
        modified_cantor_set(x_start + 2 * piece, x_start + 3 * piece, y_position - spacing, depth - 1) +
        modified_cantor_set(x_start + 4 * piece, x_end, y_position - spacing, depth - 1)
    )

    return segments + recursive

# Generate and plot
segments = np.array(modified_cantor_set(x_start, x_end, y_position, depth))

plt.figure(figsize=(25, 25))
for x_start, x_end, y in segments:
    plt.fill_between([x_start, x_end], [y]*2, [y - bar_height]*2, color="black")

plt.axis('off')
plt.show()

The image depicts the visual outcome of the modified algorithm, with each bar giving rise to a trio of narrower vertical bars in a cascading, top-down arrangement. As recursion progresses, the figure takes on a dense, columnar appearance reminiscent of tridents or stylized organ pipes, highlighting the fractalâ€™s increasing intricacy.

2.2. Random Cantor Sets

This second variation of the code introduces randomness into the Cantor Set generation process. Instead of consistently removing the same segments, each level divides the bar into five equal parts and randomly removes two of them, preserving three segments chosen at random. The recursive function then operates only on the retained segments, resulting in structural variation at every depth. This randomness introduces asymmetry and breaks the strict self-similarity of the original set, generating a more organic and irregular fractal pattern. 


import numpy as np
import matplotlib.pyplot as plt
import random

# Parameters
x_start = 0
x_end = 5
y_position = 0
bar_height = 5
spacing = 10
depth = 6  # Try smaller at first, like 4

# Randomized Cantor Set: split into 5, remove 2 at random
def random_cantor_set(x_start, x_end, y_position, depth):
    if depth == 0:
        return []

    length = x_end - x_start
    piece = length / 5

    # Generate 5 pieces (start and end x-coordinates)
    pieces = []
    for i in range(5):
        start = x_start + i * piece
        end = start + piece
        pieces.append([start, end, y_position])

    # Randomly pick 3 to keep (remove 2)
    kept_pieces = random.sample(pieces, 3)

    # Recursively generate for kept pieces
    recursive = []
    for seg in kept_pieces:
        recursive += random_cantor_set(seg[0], seg[1], y_position - spacing, depth - 1)

    return kept_pieces + recursive

# Generate and plot
segments = np.array(random_cantor_set(x_start, x_end, y_position, depth))

plt.figure(figsize=(25, 25))
for x_start, x_end, y in segments:
    plt.fill_between([x_start, x_end], [y]*2, [y - bar_height]*2, color="black")

plt.axis('off')
plt.show()

The image reflects the outcome of this randomized recursive logic. While the overall form still suggests a fractal, the pattern lacks uniformity across horizontal positions and levels, with each column displaying distinct arrangements of bar widths and spacings. The resulting structure resembles a series of abstract barcodes or digital glyphs, emphasizing variation and unpredictability within a controlled recursive framework.
